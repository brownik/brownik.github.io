우리가 지킬 최소한의 규칙 5가지

복잡한 컨벤션보다 중요한 것은, 반드시 지킬 수 있는 약속입니다.

모든 작업은 이슈 생성부터 시작한다
"티켓 없는 작업은 존재하지 않는다." 작업의 근거를 먼저 만드세요.
Master/Main 브랜치에 직접 Push 금지
운영 환경 보호를 위해 로컬에서 직접 업로드하는 것을 원천 차단합니다.
브랜치명은 이슈 번호를 포함한다
✓ feature/102-login
✗ feature/login, update-api (이슈 번호 없음)
Merge 전에는 반드시 동료의 리뷰(MR/PR)를 거친다
최소 1명 이상의 승인이 있어야 코드를 병합할 수 있습니다.
충돌(Conflict)은 MR 작성자가 해결한다
작업 내용을 가장 잘 아는 당사자가 로컬에서 해결 후 다시 Push합니다.
